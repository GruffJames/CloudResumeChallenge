Work

2019-2024

Clinisys, Software Developer-Microbiology coordinator

Member of a development team for the GLIMS product, which is a laboratory information management system (LIMS) used as a critical piece of laboratory automation infrastructure for 200+ hospitals and laboratory networks across europe. The role was initially based in Gent, Belgium before I moved to Glasgow to continue working within a team there.

Responsibilities included:

  • Collaborating with an international team of developers, QA/testers, and product analysts to develop new features as well as maintain existing features, to ensure scalability, security, and optimal performance in an agile work environment.
  • Contribute to requirements gathering and taking part in the software development lifecycle (SDLC).
  • The specialist role of microbiology coordinator, where I am the primary contact for microbiology module incidents, as well as completing the related development work.
  • Optimise installer scripts written using Powershell (Windows) and KSH shell scripting (Linux) for GLIMS development builds launched in Docker on a Virtual Machine.
  • Deliver intuitive and user-friendly interfaces using OpenEdge Ultra Controls for Microsoft .NET framework (WinForms) as well as WPF .NET with C#.
  • Design and develop SQL relational database schema and queries for an Oracle or Progress database.
  • Write code using Object oriented programming principles.
  • Review colleagues' code and advise on improvements, as well as mentoring junior colleagues to help them develop their own skills.
  • Investigate and debug issues escalated to the development team and provide technical analysis for corrections either within the code or by customer configuration.
  • This involves reading client/server log files for information, reading documentation of the product, analysing GIT logs to understand the context of past code changes,discussing the issue with customer support from the local country, and combining all that with my knowledge of the product.
  • If necessary, provide hotfixes for customers generated using Jenkins builds.
  • Create documentation in code and in Azure DevOps as part of completing work items.
  • Create unit tests to ensure future changes do not break current required functionality as well as to create new features by test-driven development (TDD).
  • Write Python tests simulating lab to lab communication and between machines such as Kiestra.

Large projects completed

Alles gurgelt”/”Everything gurgles

Worked as part of a team to create the "Alles gurgelt”/”Everything gurgles” COVID platform for Vienna in Austria. The project created a platform to give people in the region access to weekly COVID lab tests. To ensure the project was able to reach its eventual 500,000 samples a day, It required a new specimen pooling module to provide a large increase in performance to enable the massive throughput of samples required.

This involved creating:

  • A screen for specimen entry via barcodes, a query browser for viewing, processing, and generating pooling statistics as well as a number of other screens for displaying other information using OpenEdge Ultra Controls for Microsoft .NET framework.
  • Functions for the in-house scripting language MISPL.
  • Database schema and backend code to handle the above features.
  • For more information see: https://web.archive.org/web/20220914034130/https://allesgurgelt.at/wien/
    https://www.lead-horizon.com/en/faq-school-testing
Ségur numérique

Worked on the “Ségur numérique” project, where the french government mandated a large extension in compliance standards to improve security and access to healthcare data for physicians and patients. This involved:

  • Building a high performance patient search system, using a dynamically generated query.
  • Building a system to convert patient data to the new compliance standard at scale.
  • A reporting system for transferring data to a central government server.
  • For more information see:
    https://esante.gouv.fr/segur/biologie-medicale
